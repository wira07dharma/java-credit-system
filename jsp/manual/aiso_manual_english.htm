1        General Information

 

1.1      AISO Application

 

Accounting Information System Online (AISO) is an application used by Accounting Department to keep in record of every financial transaction thats going on within a company.

 

AISO devise transaction documents into financial reports which describe companys financial condition on a certain periode. These reports can be viewed or printed.

 

AISO has the ability to be accessed (online) from remote places by telephone or internet connection. This provide owners or management of a company with the ability to monitor their business from anywhere in the world as long theres a telephone or internet connection.

2.4      Journal

AISO is used to keep in record transactions such as purchasing, production, selling, inventory and also operational costs for instance transportation, electricity, telephone, water, etc to the accounting system based on the available documents.

 

2.4.1          Journal

Steps to make a journal of a transaction :

v      Click Add New Journal on the Search Journal or List Journal page.

v      Type the information needed for the process such as :

    • Voucher No, the first four number shows the year and month of the transaction. The last four number indicate the journal sequence in a periode (automatically).
    • Entry Date, the date of journal process is being done (automatically).
    • Operator Name, indicate the user whos doing the entry (automatically).
    • Currency, shows the currency used for the transaction (IDR or $US).
    • Transaction Date, the date of the transaction (based on the transaction document).
    • Contact Name, indicate the contact (can be a company or person) who involved on the transaction. The contact will be taken from master data (reference data).
    • Description, shows the note of the transaction.

 

v      After all information are inputed, click Add New Detail to add the accounts which involved on the transaction and the amount.

v      After details are inputed, there will be a red color message which saying if the journal is balance or not.

v      If the journal has balanced click Posted Journal to save the journal. Or click Back to Journal List to go back to the Journal list without saving it.

 

Editing Steps :

v      From the journal list, select the Voucher No which going to be edited.

v      If you only want to edit the journal header, do the editing then click Posted Journal to save the changes.

v      If you want to change the journal detail, click the account that you want to edit then do the editing. Then click Update Journal. After the journal has balanced, click Posted Journal to save the changes.

 

Deleting Steps :

v      From the journal list, select the Voucher No, which to be deleted

v      Click Delete Journal, afterward there will be a confirmation note which said Are you sure to delete journal ?. Click Yes Delete Journal to delete the journal or click Cancel to cancel deleting.

 

Below is the steps of making, editing and deleting a journal of an account which has sub ledger.

 

2.4.2          Sub Ledger

In this module the detail of transaction data which has relationship with fixed/other asset, receivable and payable will be recorded. Accounts having sub ledger are defined in the Account Chart sub module.

 

Steps of journaling :

v      The preliminary steps is exactly like the journal process above.

v      During journaling process, if an account has to have sub ledger entry, you will be asked to input detail information in the sub ledger regarding the transaction.

 

Sub Ledger Receivable/Payable

Add these following data if receivable/payable increase :

    • Billing Number, specify the receivable/payable transaction number.
    • Due Date, specify the due date of the receivable/payable
    • Amount , specify the amount of receivable/payable.

 

Add these following data if receivable/payable decrease :

    • Billing Number, specify the receivable/payable transaction number.
    • Paid Amount, specify the amount of receivable/payable being paid. Click reference to get the related receivable/payable.

 

Sub Ledger Fixed Asset/Other Asset

Add these following data if fixed/other asset is debeted:

    • Asset Status, indicate the status of an asset (Buying).
    • Asset Group, indicate the group of asset. Choose from the list.
    • Asset Name, indicate name of asset.
    • Asset No, indicate number of asset in its group.
    • Type, indicate the type of depreciation or amortization used for the asset.
    • Owns Date, indicate the date of the asset being in possesion.
    • Owns Cost, indicate the asset buying cost.
    • Residue, shows the assets salvage value if the usable time has ended.
    • Usable Time, indicate period(yearly) of how long the asset can use.

 

Add these following data if fixed/other asset is credited :

    • Asset Status, indicate the status of an asset (Out of Date).
    • Click get asset to get the related asset.
    • Sold/Damage Value, indicate amount of transaction if asset is sold or damage.

 

v      Click Back to Journal to continue jurnaling process.

v      If the journal has balanced, then it can be posted by clicking Posted Journal.

v      After posted, the journal will be shown up in the journal list page.

 

 

2.5      Report

This module provides financial reports which can be accessed by clicking the Report button in the main page. The reports can be viewed by installing Adobe Acrobat Reader application or other applications which can read a .pdf file format.

 

2.5.1          General Ledger

This module provide General Ledger. To view this report for an account, do the following steps :

v      Select the chosen account. Note : only postable account has general ledger.

v      Select the chosen periode.

v      Select the chosen currency.

v      Click Preview Report, the the report will be viewed on a new window. From this window, you can save the report to a disk or print it.

 

General Ledger report consist of two part which is :

v      Summary, this part point up the general information of general ledger for instance previous balance (opening balance), current balance, and mutation during a periode chosen.

v      Detail, this part viewing detail information about mutation on the chosen periode.

 

2.5.2          Query Journal

Through this module, you can view journal report based on several parameters. The steps are :

v      Define the selected journals transaction date.

v      Define the selected journals voucher number (optional)

v      Define the selected journals invoice (optional)

v      Define the selected journals contact name (optional)

v      Define the operator of the selected journal.

v      Define the currency.

v      Select the sorting type.

v      Click Preview Report, to view the journal and then you can either save it to a disk or print it.

This report is viewing journals detail information such as : voucher number, entry date, transaction date, operator, contact name, description, and accounts connected with the journal.

 

2.5.3          Trial Balance

This module present the Trial Balance report. To view the reports do these following steps :

v      Define the selected date.

v      Define the currency.

v      Click Preview Report, to view the trial balance report based on the chosen parameters.

This report is showing the previous balance, mutation, and current balance of all postable account per the chosen date.

 

2.5.4          Cash Flow

This module present the Cash Flow report, which showing cash previous balance, cash in, cash out, and cash current balance. To view the reports do these following steps :

v      Select the chosen cash flow report type (periodic or weekly)

v      Select the chosen periode (If the chosen report type is periodic).

v      Define the chosen time range (If the chosen report type is weekly).

v      Select the currency.

v      Click Preview Report, to view the cash flow report based on the chosen parameters. The report will be shown on a new window from which you can either save it to a disk or print it.

 

2.5.5          Gross Profit of Goods Sold

Through this module you can view the Gross Profit of Goods Sold (sales information, cost of goods sales, and gross profit of a product). To view the reports do these following steps :

v      Select one of the sales account or all sales to view the report.

v      Select the chosen periode.

v      Select the currency.

v      Click Preview Report, to view the Gross Profit of Goods Sold report based on the chosen parameters. The report will be shown on a new window from which you can either save it to a disk or print it.

2.6      Periode

Through this module you can set new accounting periode (Setup Periode) and close the current periode (Close Book). This modul can be accessed by clicking the Periode button from the main menu.

 

2.6.1          Setup Periode

Actually AISO will automatically creating a periode in the sub module Application Setting (System). The purpose of this module is only to define additional information for the periode such as :

v      Last entry, indicate the last date to entry data for the related periode.

v      Name, indicate the name/title of the periode.

v      Description.

Periode editing can be done by clicking the Period Name to open the periode editor form. Do the editing and afterward click Save, to save the changes.

 

 

2.6.2          Close Book

On the end of an accounting periode, closing the periode must be done (close book) in order to cut off the transactions for a periode. If the close book is done, all of the transactions in that periode can only be viewed (read only). The close book process will automatically creating a new accounting periode.

 

To do the closing, click Close Book, then a confirmation note will appear saying : Are sure the following processes already done ?. Click Yes Close Book to close aperiode or Cancel to cancel it.

2.7.1.1         Account Chart

In this menu, accounts for the accounting system is recorded. Accounts are classified into 8 groups, they are :

v      Activa : account number start with number 1

v      Liability : account number start with number 2

v      Equity : account number start with number 3

v      Income : account number start with number 4

v      Cost of Goods Sold : account number start with number 5

v      Expenses : account number start with number 6

v      Other income : account number start with number 7

v      Other expenses : account number start with number 8

 

 

To create a new account, first you must identify the account group in which this account belong to, afterward do these following steps :

v      Click Add New Account.

v      Then input the account information such as :

         Account reference, specify the group of the account.

         Account name,

         Account number, the first number is set automatically based on the account group then followed by account number which you can use your own company account number format.

         Level, AISO provide 6 account level where level 1 is the header of level 2, level 2 is the header for level 3, and so on.

         Normal Sign, specify the account increment is on debet side or credit.

         Postable, specify if the account is used in the journal process or not. You can pick yes or no.

         Subledger, specify if the account has a sub ledger or not. Note : in AISO there are 4 sub ledgers available :

o        fixed asset sub ledger, to record detail information on fixed asset.

o        other asset sub ledger, to record detail information on other asset.

o        Receivable sub ledger, to record detail information on receivable.

o        Liability / payable sub ledger, to record detail information on liabilities.

         For a postable account (postable=yes), input the beginning balance of the account.

v      After all the information are inputed, click Save Account to save the record.

 

Editing an account :

v      Select the account

v      Do the editing

v      Click Save Account to save the changes.

 

 

Deleting an account :

v      Select the account

v      Click Delete Account, then a confirmation message will appear saying Are you sure delete ?. Click Yes Delete to continue deleting or click Cancel to cancel it.

 

After concluding the process above, click Upload Account Chart to activate the account with the latest detail/information.

 

Viewing and Printing Account :

v      Click List Account, then a new window will appear containing list of the chosen account.

v      To print it, on the menu click File then Print.

 

 

2.7.1.2         Account Link

This module is used to define the relationship between an account to the other, vis--vis in reporting manner. Types of account link :

 

v      Cash Flow :

In this menu, accounts which considered to be cash are identified. This account link is required in creating Cash Flow Report, therefore it must be completed. Accounts that appear on the report are the opponent of accounts which are defined here based on the available journal.

 

v      Gross Profit of Goods Sold :

In this menu, the pair/opponent of sales and cost of goods sold accounts are identified. This account link is used in creating Gross Profit of Goods Sold Report. Profit/Loss is the substraction of sales by cost of good sold. This account link is optional, meaning you can use it or not based on your needs.

 

v      Corporation Income Tax :

In this menu, accounts that associated with income tax are identified. This account link will affect Profit/Loss Report therefore it must be completed.

 

v      Current Earning Year :

In this menu, accounts that associated with Current Earning Year are identified. This account link will record the profit accumulation from the first accounting periode to the latest. This record will appear on and affected the Balance Sheet Report , therefore it must be completed.

 

v      Current Earning Period (Laba/Rugi Periode Berjalan) :

Here, accounts that associated with Current Earning Period are identified. This account link will record the profit/loss per accounting periode. This record will appear on and affected the Profit Loss Report , therefore it must be completed.

 

v      Change Withdrawal :

Here, accounts that associated with Change Withdrawal are identified. This is a temporary account which will be used in the process of Special Jurnal for instance when there are an occation needs exchanging US dollar into Rupiah. This account link will record the profit/loss per accounting periode. This account link is optional, meaning you can use it or not based on your needs.

 

v      Fixed Asset :

Here, accounts that associated with Fixed Asset are identified. This account link is associated with other modules regarding fixed asset management (Asset Group and Depreciation report), therefore it must be completed.

 

v      Other Asset :

Here, accounts that associated with Other Asset are identified. This account link is associated with other modules regarding other asset management (Asset Group and Amortization report), therefore it must be completed.

User Management is used for managing privilege in using AISO application. This menu is divided into several sub menus :

 

o        User

Here, users of application are listed. Employee that are given privilege to access this application will have their own user ID and password to log in the application. User will be grouped according to their authorization.

v      Creating new user

Click Add New User to open user management form editor in which user information are inputed. Each user will have his/her own privileges based on the chosen Group Assigned. Afterward, click Save User to save the information.

 

v      Editing

Click the login ID of chosen User to open user management form editor then do the editing. Click Save User to save the changes.

 

v      Deleting

Click the login ID of chosen User to open user management form editor, then click Delete User. A confirmation message will appear saying : Are you sure to delete ?. Click Yes Delete User to continue deleting. Click Back to User List to go back to user list.

 

 

o        Group

Group is used to classify privileges. Each user will have one or more access group based on his/her authorization.

v      Creating new group

Click Add New Group to open group management form editor in which group information are inputed. Each group will have his/her own privileges to access a module based on the chosen Privilege Assigned. Afterward, click Save Group to save the information.

 

v      Editing

Click the group name of chosen Group to open group management form editor then do the editing. Click Save Group to save the changes.

 

v      Deleting

Click the group name of chosen Group to open group management form editor, then click Delete Group. A confirmation message will appear saying : Are you sure to delete ?. Click Yes Delete Group to continue deleting. Click Back to Group List to go back to group list.

 

 

o        Privilege

This sub menu is used to make access permission to each module in AISO application.

v      Creating new privilege

Click Add New Privilege to open privilege management form editor in which privilege information are inputed. Each privilege will have his/her own privileges to access a module based on the chosen Privilege Assigned. Afterward, click Save Privilege to save the information.

 

v      Editing

Click the privilege name of chosen Privilege to open privilege management form editor then do the editing. Click Save Privilege to save the changes.

 

v      Deleting

Click the privilege name of chosen Privilege to open privilege management form editor, then click Delete Privilege. A confirmation message will appear saying : Are you sure to delete ?. Click Yes Delete Privilege to continue deleting. Click Back to Privilege List to go back to privilege list.

2.8.2          System

o        Application Setting

Use this menu to set up the periode variable used by AISO.

v      Interval Periode, is a span of time between one periode to the next in months. Interval periode = 1, meaning the periode used in month.

 

v      Last entry duration, is a span of time between the last date on a periode with the last date to entry in days. Last Entry Duration = 15, meaning the span of time given to entry data is 15 days start from the last date on the periode.

 

Click Load New Value to have the last setting of Interval Periode and Last Entry Duration. To update or edit the setting, click the variable name then do the changes. Afterward click Update Value.

 

o        Back Up Service

This service is used to run or stop service which doing database back-up. AISO will automatically doing the back-up process according to the configurations:

v      Start Time, indicate starting time to back-up

v      Periode, indicate the span of time of back-up periode.

v      Source Database, indicate the path to the databse which will be back-up.

v      Target 1, indicate traget path 1 where back-up database will be kept.

v      Target 2, indicate traget path 1 where back-up database will be kept.

v      Target 3, indicate traget path 1 where back-up database will be kept.

 

Commands used in this module are :

v      Start, is used to start database back-up process based on the configuration.

v      Stop, is used to stop database back-up process.

v      Update Configuration, is used to update configuration data.

v      List Log, is used in viewing configuration data.
